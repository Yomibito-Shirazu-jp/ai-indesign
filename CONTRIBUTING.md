# 開発にご協力いただける方へ（CONTRIBUTING）

AI in Design（旧称: indesign-uxp-server）をご覧いただき、ありがとうございます！
このプロジェクトは、**「日本の美しい文字組みを、もっと簡単に、もっと安全に」** という思いのもと、誰もが自由に使える（MITライセンス）オープンソースとして公開されています。

プログラマーの方はもちろん、**「コードは書けないけれど、印刷屋としての知見がある」「DTPの現場でこういう機能が欲しい」**といったデザイナー・DTPオペレーターの方からのご意見も大歓迎です！

---

## 💡 どうやって参加できるの？

### 1. バグ報告・機能リクエスト（誰でも大歓迎！）
「ここがおかしい」「こんな機能があればもっと便利なのに！」と思ったら、お気軽にGitHubの **Issues（イシュー）** からお知らせください。

**【報告テンプレートの例】**
- **こんな問題が起きました（バグ報告）**：どんな操作をして、どうなってしまったか、エラーの内容などを教えてください。
- **こんな機能が欲しい（機能要望）**：印刷現場での具体的な困りごとや、「〇〇を自動化したい」という要望を自由に書いてください。

### 2. プログラムの追加・修正（エンジニア・開発者向け）
JavaScriptやNode.jsが書ける方は、ぜひ直接プログラムの改善にご協力ください！特定の作業を自動化するための「新しいツール」の追加も大歓迎です。

## 💻 開発環境の作り方（エンジニア向け）

このツールを開発・改造するための手順です。

### 必須環境
- **Adobe InDesign**: 動作確認のために必要です。
- **Node.js（バージョン18以上）**: 裏側で動くサーバー用です。
- **Adobe UXP Developer Tool**: 作ったプラグインをInDesignに読み込ませるための公式ツールです。
- **Git**: ソースコードの管理用です。

### セットアップ手順

1. **このリポジトリをパソコンにダウンロード（クローン）します**
   ```bash
   git clone https://github.com/<your-org>/ai-in-design.git
   cd ai-in-design
   ```

2. **必要なパッケージをインストールします**
   ```bash
   npm install
   ```
   ※ `bridge` フォルダの中にもサーバーがありますので、そこでも実行してください。
   ```bash
   cd bridge && npm install && cd ..
   ```

3. **InDesignでプラグインを読み込む**
   `Adobe UXP Developer Tool` を開き、このフォルダ内の `plugin/manifest.json` を **「Load and Watch」** で読み込んでください。

4. **テスト用サーバーを立ち上げる**
   ```bash
   npm run dev
   ```
   ※ ブリッジサーバー（`bridge/server.js`）はMCPサーバー起動時に自動で立ち上がります。手動で起動する必要はありません。

## 🛠 新しい「ツール（機能）」を追加するには？

AIにお願いできる新しい機能を増やしたい場合は、以下の手順で追加できます。

1. **処理（ハンドラ）を追加する**
   `src/handlers/` の中にある適切なファイル（または新しいファイル）に、InDesignを操作する処理をJavaScriptで書きます。

2. **ツールをAIに教える（定義を追加する）**
   `src/types/` の中にある定義ファイルに、「このツールはこういう名前で、こういう指示を受け取ります」という説明書きを追加します。ここをわかりやすく書くことで、AIが正しくツールを使ってくれるようになります。

3. **サーバーに登録する**
   `src/index.js` にて、新しく作った処理を登録・紐付けします。

> 💡 日本語処理系のコード構成については、[README.md の開発者向けセクション](README.md#-開発者向け日本語縦書き対応のコード構成について) にファイル名と機能の対応表があります。

## 🧪 テストについて

機能を追加したり直したりした場合は、おかしくなっていないかテストで確認をお願いします。

⚠️ **テスト実行の前提条件**：InDesignが起動していて、UXP Developer Tool経由でプラグインが読み込まれている（ブリッジサーバーと接続されている）状態が必要です。

```bash
npm test
```
